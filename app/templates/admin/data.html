How it works

  • Skip existing: a user row matching an existing mobile or username is skipped; a scheme row matching an existing name is skipped. No duplicates are created.
  • Passwords: set each new user's password in the password column. (Exports leave it blank — fill it in before re-importing to a new instance.)
  • Yes/No columns (BPL card, disability, bank, etc.) accept true/false (or yes/no, 1/0).
